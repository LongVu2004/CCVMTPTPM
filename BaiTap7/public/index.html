Kiến Trúc CI/CD

⚙️

Continuous Integration

Tự động hóa quá trình tích hợp mã nguồn.

  • Build source code tự động
  • Chạy Unit/Integration Tests
  • Kiểm tra chất lượng code (Linting)
🚀

Continuous Deployment

Tự động hóa quy trình phân phối sản phẩm.

  • Deploy thẳng lên Production/Pages
  • Không thời gian chết (Zero-downtime)
  • Sẵn sàng Rollback khi có sự cố

Quy Trình Hoạt Động (Workflow)

  1. Commit & Push Code

    Developer đẩy mã nguồn mới lên Repository gốc.

  2. Trigger Pipeline

    Hệ thống tự động đọc file cấu hình .yml và tạo phiên chạy mới.

  3. Execute Jobs (Runner)

    Các máy chủ ảo (Runner) nhận lệnh và thực thi tuần tự các stage: Build ➔ Test ➔ Deploy.

  4. Publish Artifacts

    Sản phẩm đã biên dịch được đóng gói và đẩy lên môi trường Hosting tĩnh (Pages).

Thuật Ngữ Cốt Lõi

.gitlab-ci.yml File "bộ não" điều khiển toàn bộ kịch bản CI/CD.
Pipeline Chuỗi dây chuyền sản xuất tự động gồm nhiều công đoạn.
Stage / Job Nhóm nhiệm vụ (Stage) và các tác vụ chi tiết (Job) bên trong.
Shared Runner Máy chủ do hệ thống cung cấp miễn phí để chạy script.
